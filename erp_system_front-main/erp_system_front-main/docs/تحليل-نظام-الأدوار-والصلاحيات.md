# تحليل شامل لنظام الأدوار والصلاحيات (RBAC)

## 🎯 نظرة عامة

نظام **Role-Based Access Control (RBAC)** متقدم ومرن يدير صلاحيات المستخدمين في النظام. يتميز بـ:
- ✅ **نظام هرمي** - أدوار مع أولويات مختلفة
- ✅ **صلاحيات ذكية** - منطق هرمي للصلاحيات (manage → edit → view)
- ✅ **صلاحيات مباشرة** - تجاوز الأدوار لمستخدمين محددين
- ✅ **صلاحيات مؤقتة** - انتهاء صلاحية تلقائي
- ✅ **حماية من التعديل** - صلاحيات وأدوار النظام محمية
- ✅ **تسجيل شامل** - Audit logs لكل تغيير

## 🏗️ البنية الأساسية

### **1. الجداول في قاعدة البيانات**

```
User (المستخدمين)
  ├─ UserRole (أدوار المستخدمين)
  │    └─ Role (الأدوار)
  │         └─ RolePermission (صلاحيات الأدوار)
  │              └─ Permission (الصلاحيات)
  │
  └─ UserPermission (صلاحيات مباشرة للمستخدمين)
       └─ Permission (الصلاحيات)

PermissionLog (سجل التغييرات)
```

### **2. الجداول التفصيلية**

#### **Permission (الصلاحيات)**
```typescript
{
  id: string (cuid)
  resource: string        // مثل: "dashboard.users"
  action: string         // مثل: "view", "create", "edit"
  displayName: string    // الاسم المعروض بالعربية
  description: string    // وصف الصلاحية
  category: string       // التصنيف: "إدارة المستخدمين"
  conditions: JSON       // شروط إضافية (اختياري)
  isSystem: boolean      // صلاحية نظام (محمية)
}
```

**مثال**:
```json
{
  "resource": "dashboard.trainees",
  "action": "create",
  "displayName": "إضافة متدربين",
  "description": "إضافة متدربين جدد للنظام",
  "category": "إدارة المتدربين",
  "isSystem": false
}
```

#### **Role (الأدوار)**
```typescript
{
  id: string (cuid)
  name: string           // مفتاح فريد: "admin", "manager"
  displayName: string    // "مدير النظام", "مدير"
  description: string    // وصف الدور
  color: string          // لون hex للعرض
  icon: string           // أيقونة FontAwesome
  priority: number       // الأولوية (1000 أعلى)
  isSystem: boolean      // دور نظام (محمي)
  isActive: boolean      // نشط/معطل
}
```

#### **UserRole (ربط المستخدم بالأدوار)**
```typescript
{
  userId: string
  roleId: string
  assignedBy: string     // من قام بالتعيين
  assignedAt: DateTime
  expiresAt: DateTime?   // انتهاء صلاحية (اختياري)
  conditions: JSON?      // شروط إضافية
  isActive: boolean
}
```

#### **RolePermission (ربط الدور بالصلاحيات)**
```typescript
{
  roleId: string
  permissionId: string
  granted: boolean       // ممنوح/محظور
  expiresAt: DateTime?
}
```

#### **UserPermission (صلاحيات مباشرة للمستخدم)**
```typescript
{
  userId: string
  permissionId: string
  granted: boolean       // ممنوح/محظور
  assignedBy: string
  reason: string?        // سبب المنح المباشر
  expiresAt: DateTime?
}
```

## 🧠 المنطق الذكي للصلاحيات

### **1. الصلاحيات الهرمية**

```typescript
// إذا كان لديه "manage" → لديه كل شيء
hasPermission('dashboard.users', 'manage') 
  ⇒ true لـ view, create, edit, delete, export

// إذا كان لديه "edit" → لديه "view"
hasPermission('dashboard.users', 'edit')
  ⇒ true لـ view

// إذا كان لديه "create" → لديه "view"
hasPermission('dashboard.users', 'create')
  ⇒ true لـ view

// إذا كان لديه "delete" → لديه "view"
hasPermission('dashboard.users', 'delete')
  ⇒ true لـ view
```

**الهرمية**:
```
manage (الأعلى)
  ├── create
  ├── edit
  ├── delete
  ├── export
  └── view (الأدنى)
```

### **2. أولوية الصلاحيات**

```
1. الصلاحيات المباشرة (UserPermission) ← **أولوية قصوى**
2. صلاحيات الأدوار (RolePermission) ← أولوية عادية
```

**مثال**:
```typescript
// المستخدم لديه:
- Role: manager → يمنح dashboard.trainees.edit
- UserPermission مباشرة: dashboard.trainees.edit = false (محظور)

// النتيجة: false (الصلاحية المباشرة تجاوز الدور)
```

### **3. الأدوار الخاصة**

```typescript
// Super Admin - لديه كل شيء دائماً
if (userRoles.includes('super_admin')) {
  return true; // لأي صلاحية
}

// Admin - لديه معظم الصلاحيات
if (userRoles.includes('admin')) {
  return true; // لمعظم الصلاحيات
}
```

## 🔐 الحماية والأمان

### **1. حماية أدوار النظام**

```typescript
// لا يمكن حذف أدوار النظام
if (role.isSystem) {
  throw new ForbiddenException('لا يمكن حذف أدوار النظام');
}

// لا يمكن إلغاء تفعيل أدوار النظام
if (role.isSystem && updateDto.isActive === false) {
  throw new ForbiddenException('لا يمكن إلغاء تفعيل أدوار النظام');
}

// لا يمكن تعديل صلاحيات super_admin
if (role.name === 'super_admin') {
  throw new ForbiddenException('لا يمكن تعديل صلاحيات مدير النظام الرئيسي');
}
```

### **2. حماية الصلاحيات النظام**

```typescript
// لا يمكن تعديل صلاحيات النظام
if (permission.isSystem) {
  throw new ForbiddenException('لا يمكن تعديل صلاحيات النظام');
}

// لا يمكن حذف صلاحيات النظام
if (permission.isSystem) {
  throw new ForbiddenException('لا يمكن حذف صلاحيات النظام');
}
```

### **3. صلاحيات النظام (isSystem: true)**

هذه الصلاحيات لا يمكن تعديلها أو حذفها:
- `dashboard.permissions.view` - عرض الصلاحيات
- `dashboard.permissions.manage` - إدارة الصلاحيات

## 🔧 كيفية عمل النظام

### **Backend (NestJS)**

#### **1. Decorators للحماية**

```typescript
// في Controller
@Controller('trainees')
@UseGuards(JwtAuthGuard, PermissionGuard)
export class TraineesController {
  
  // صلاحية واحدة
  @Get()
  @RequirePermission('dashboard.trainees', 'view')
  async getAll() { /* ... */ }
  
  // صلاحيات متعددة (أي واحدة)
  @Post()
  @RequireMultiple({
    permissions: [
      { resource: 'dashboard.trainees', action: 'create' }
    ],
    roles: ['admin', 'manager'],
    operator: 'OR' // أي واحدة تكفي
  })
  async create() { /* ... */ }
  
  // دور محدد
  @Delete(':id')
  @RequireRole('admin')
  async delete() { /* ... */ }
  
  // أي دور من قائمة
  @Put(':id')
  @RequireAnyRole('admin', 'manager')
  async update() { /* ... */ }
}
```

#### **2. Guards (الحراس)**

**PermissionGuard** - يفحص الصلاحيات:
```typescript
async canActivate(context: ExecutionContext): Promise<boolean> {
  // 1. استخراج المستخدم من الـ request
  const user = request.user;
  
  // 2. جلب صلاحيات المستخدم من قاعدة البيانات
  const userPermissions = await permissionsService.getUserPermissions(user.userId);
  
  // 3. فحص الصلاحية المطلوبة
  if (!userPermissions.hasPermission(resource, action)) {
    throw new ForbiddenException('ليس لديك صلاحية');
  }
  
  return true;
}
```

#### **3. PermissionsService - الوظائف الأساسية**

```typescript
// جلب صلاحيات مستخدم
async getUserPermissions(userId: string): Promise<UserPermissions> {
  // 1. جلب أدوار المستخدم
  const userRoles = await getUserRoles(userId);
  
  // 2. جمع صلاحيات الأدوار
  const rolePermissions = collectRolePermissions(userRoles);
  
  // 3. جلب الصلاحيات المباشرة
  const directPermissions = await getDirectPermissions(userId);
  
  // 4. دمج (الصلاحيات المباشرة تجاوز الأدوار)
  const finalPermissions = merge(rolePermissions, directPermissions);
  
  // 5. إرجاع مع دوال المساعدة
  return {
    roles: [...],
    permissions: {...},
    hasPermission: (resource, action) => {
      // منطق الفحص الذكي
    },
    hasRole: (roleName) => {...},
    hasAnyRole: (roleNames) => {...}
  };
}
```

## 🔄 سير العمل (Workflow)

### **سيناريو 1: تعيين دور لمستخدم جديد**

```
1. إنشاء مستخدم جديد
   POST /users
   
2. تعيين دور له (مثلاً: manager)
   POST /permissions/assign-role
   Body: {
     userId: "user_id",
     roleId: "manager_role_id"
   }
   
3. النتيجة:
   - المستخدم الآن لديه جميع صلاحيات المدير
   - يمكنه الوصول لصفحات المتدربين والبرامج
   - لا يمكنه إدارة الصلاحيات
```

### **سيناريو 2: منح صلاحية إضافية مباشرة**

```
1. المستخدم لديه دور "employee"
   → صلاحيات محدودة (view, create, edit للمتدربين)
   
2. نريد منحه صلاحية إضافية مؤقتة:
   POST /permissions/assign-permission
   Body: {
     userId: "user_id",
     permissionId: "delete_trainees_permission_id",
     granted: true,
     reason: "حذف متدربين منتهي صلاحيتهم",
     expiresAt: "2025-12-31T23:59:59Z"
   }
   
3. النتيجة:
   - الآن لديه صلاحية حذف المتدربين
   - الصلاحية تنتهي في 31 ديسمبر 2025
   - يمكن تتبع السبب في سجل الأنشطة
```

### **سيناريو 3: حظر صلاحية من دور**

```
1. المستخدم لديه دور "manager"
   → يشمل dashboard.trainees.delete
   
2. نريد منعه من الحذف (لكن بقية الصلاحيات تبقى):
   POST /permissions/assign-permission
   Body: {
     userId: "user_id",
     permissionId: "delete_trainees_permission_id",
     granted: false,  // ⚠️ حظر
     reason: "تقييد مؤقت"
   }
   
3. النتيجة:
   - لا يمكنه حذف المتدربين
   - باقي صلاحيات المدير تعمل عادياً
   - الصلاحية المباشرة (false) تجاوزت الدور
```

## 🔍 التشخيص والأخطاء

### **مشكلة: "ليس لديك صلاحية"**

**الحل**:
```sql
-- 1. التحقق من أدوار المستخدم
SELECT ur.*, r.displayName 
FROM user_roles ur
JOIN roles r ON ur.roleId = r.id
WHERE ur.userId = 'user_id' AND ur.isActive = 1;

-- 2. التحقق من صلاحيات الدور
SELECT rp.*, p.resource, p.action, p.displayName
FROM role_permissions rp
JOIN permissions p ON rp.permissionId = p.id
WHERE rp.roleId = 'role_id' AND rp.granted = 1;

-- 3. التحقق من الصلاحيات المباشرة
SELECT up.*, p.resource, p.action
FROM user_permissions up
JOIN permissions p ON up.permissionId = p.id
WHERE up.userId = 'user_id';
```

### **مشكلة: super_admin لا يمكنه تعديل الصلاحيات**

**السبب**: صلاحيات super_admin محمية

**الحل**: استخدم مستخدم آخر أو عدّل الكود مؤقتاً
