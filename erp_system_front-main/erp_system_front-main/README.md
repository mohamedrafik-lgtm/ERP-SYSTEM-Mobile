# Tiba Training Center Web Application

تطبيق ويب شامل لمركز تيبا للتدريب يتضمن إدارة المتدربين والبرامج والأخبار والوظائف.

## تحليل المشروع

### فكرة المشروع
المشروع عبارة عن نظام متكامل لإدارة معهد تدريبي، يتيح إدارة المتدربين، البرامج التدريبية، المحتوى التعليمي، الحضور والغياب، الشؤون المالية، وإدارة المستخدمين والصلاحيات.

### التقنيات المستخدمة
- **الواجهة الأمامية (Frontend)**: Next.js, React, TailwindCSS
- **الخادم الخلفي (Backend)**: NestJS
- **قاعدة البيانات**: MySQL مع Prisma ORM
- **إدارة المستخدمين**: نظام الصلاحيات والأدوار المخصصة

### قاعدة البيانات
تضم الكيانات الرئيسية التالية:
- **المستخدمين (User)**: إدارة مستخدمي النظام بأدوار مختلفة
- **المتدربين (Trainee)**: بيانات المتدربين ومعلوماتهم الشخصية والتعليمية
- **البرامج التدريبية (TrainingProgram)**: البرامج المتاحة للمتدربين
- **المحتوى التدريبي (TrainingContent)**: محتويات البرامج التدريبية
- **الحضور (AttendanceRecord)**: تسجيل حضور وغياب المتدربين
- **الشؤون المالية**: نماذج للرسوم (TraineeFee) والمدفوعات (TraineePayment) والمعاملات المالية
- **بنك الأسئلة (Question)**: إدارة أسئلة الاختبارات
- **الوظائف (Job)** و **الأخبار (News)**: لنشر فرص العمل والأخبار

### الوظائف الرئيسية

#### واجهة المستخدم الأمامية
1. **الصفحة الرئيسية**: عرض معلومات عن المعهد والبرامج المتاحة
2. **التسجيل**: نموذج تسجيل للمتدربين الجدد
3. **لوحة التحكم**: واجهات إدارية للمستخدمين بحسب صلاحياتهم
4. **تصفح البرامج**: عرض تفاصيل البرامج التدريبية
5. **الوظائف والأخبار**: عرض فرص العمل والأخبار

#### لوحة التحكم
1. **إدارة المتدربين**: إضافة، تعديل، حذف، وعرض بيانات المتدربين
2. **إدارة البرامج**: إنشاء وتعديل البرامج التدريبية
3. **إدارة المحتوى التدريبي**: إضافة وتنظيم المحتوى التعليمي
4. **تسجيل الحضور**: متابعة حضور المتدربين
5. **إدارة الاختبارات**: إنشاء وإدارة بنك الأسئلة
6. **الشؤون المالية**: إدارة الرسوم والمدفوعات
7. **طباعة الكارنيهات**: إصدار بطاقات للمتدربين
8. **إدارة المستخدمين والصلاحيات**: تحديد أدوار المستخدمين وصلاحياتهم
9. **سجل العمليات (Audit Log)**: تتبع العمليات التي تتم في النظام

#### الواجهات الخلفية (APIs)
يوفر النظام واجهات برمجية (APIs) لكل وظيفة رئيسية:
- التحقق والتفويض (Auth)
- إدارة المتدربين
- إدارة البرامج والمحتوى التعليمي
- تسجيل الحضور
- إدارة الاختبارات
- إدارة الشؤون المالية
- إدارة المستخدمين والصلاحيات
- إدارة الوظائف والأخبار

## التثبيت والإعداد

### 1. تثبيت التبعيات

```bash
npm install
```

### 2. إعداد ملفات البيئة

```bash
# إنشاء ملف البيئة للواجهة الأمامية
cp env.example .env.local

# إنشاء ملف البيئة للخادم الخلفي
cd backend && cp env.example .env && cd ..
```

### 3. تكوين متغيرات البيئة

#### للواجهة الأمامية (.env.local):
```env
NEXT_PUBLIC_API_BASE_URL=http://localhost:4000/api
NEXT_PUBLIC_SERVER_BASE_URL=http://localhost:4000
```

#### للخادم الخلفي (backend/.env):
```env
PORT=4000
FRONTEND_URL=http://localhost:3000
SERVER_URL=http://localhost:4000
DATABASE_URL="file:./dev.db"
JWT_SECRET=your-super-secret-jwt-key-change-this-in-production
JWT_EXPIRES_IN=7d
NODE_ENV=development
```

### 4. إعداد قاعدة البيانات

```bash
# توليد عميل Prisma
npx prisma generate

# تشغيل الترحيلات
npx prisma migrate dev

# إنشاء المستخدم الافتراضي
npx prisma db seed
```

### 5. تشغيل التطبيق

```bash
# تشغيل الخادم الخلفي
cd backend && npm run start:dev

# في نافذة طرفية جديدة، تشغيل الواجهة الأمامية
npm run dev
```
