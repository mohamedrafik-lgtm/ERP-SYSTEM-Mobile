# نظام ربط منصة المتدربين بمواعيد السداد

## نظرة عامة

تم تطوير نظام متكامل لربط منصة المتدربين بنظام مواعيد سداد الرسوم، حيث يتم:
- **إيقاف المنصة** تلقائياً عند تجاوز موعد السداد النهائي (إذا كان مفعل في الإجراءات)
- **عرض إشعارات** للرسوم المستحقة قريباً
- **مراعاة الاستثناءات** المخصصة للمتدربين

## كيف يعمل النظام

### 1. عند تسجيل دخول المتدرب:

```typescript
// في layout.tsx
useEffect(() => {
  // 1. فحص حالة الدفع من API
  const paymentStatus = await checkPaymentStatus()
  
  // 2. إذا محجوب → إعادة توجيه فورية
  if (!paymentStatus.canAccess) {
    router.push('/trainee-dashboard/blocked')
  }
}, [])
```

### 2. منطق الفحص في Backend:

```typescript
// في TraineePaymentStatusService

checkTraineePaymentStatus(traineeId) {
  
  // 1. جلب الرسوم غير المدفوعة
  const unpaidPayments = await getUnpaidPayments(traineeId)
  
  // 2. لكل رسم:
  for (payment of unpaidPayments) {
    
    // أ. جلب جدول موعد السداد
    const schedule = payment.fee.paymentSchedule
    
    if (!schedule || !schedule.actionEnabled) continue
    
    // ب. البحث عن استثناء للمتدرب
    const exception = findException(traineeId, payment.feeId)
    
    // ج. تحديد الموعد النهائي الفعلي
    const deadline = exception?.customFinalDeadline || schedule.finalDeadline
    
    // د. فحص الحالة
    if (now > deadline) {
      // 🔴 متأخر
      
      if (schedule.nonPaymentActions.includes('DISABLE_PLATFORM')) {
        return {
          status: 'BLOCKED',
          canAccessPlatform: false,
          overduePayments: [...]
        }
      }
    }
    else if (now >= schedule.paymentStartDate && now <= deadline) {
      // 🟡 مستحق قريباً
      
      return {
        status: 'PAYMENT_DUE',
        canAccessPlatform: true,
        upcomingPayments: [...]
      }
    }
  }
  
  // ✅ لا مشاكل
  return { status: 'ACTIVE' }
}
```

### 3. أولوية الاستثناءات:

```
1. استثناء خاص بالرسم (traineeId + feeId)  ← أولوية قصوى
2. استثناء عام (traineeId + feeId=null)      ← أولوية متوسطة  
3. موعد السداد العام (FeePaymentSchedule)   ← افتراضي
```

## سيناريوهات الاستخدام

### سيناريو 1: رسوم مستحقة قريباً ✅

```
المدير المالي يحدد:
- موعد سداد: 1 ديسمبر - 31 ديسمبر
- فترة سماح: 7 أيام
- الموعد النهائي: 7 يناير
- الإجراء: إيقاف المنصة ✓

المتدرب يسجل الدخول في 20 ديسمبر:
→ يرى banner أزرق
→ "لديك رسوم التسجيل مستحقة - 15,000 ج.م - متبقي 18 يوم"
→ يمكنه استخدام المنصة عادياً
```

### سيناريو 2: تجاوز الموعد + إيقاف مفعل 🔴

```
المتدرب يسجل الدخول في 10 يناير:
→ تجاوز الموعد النهائي (7 يناير) بـ 3 أيام
→ الإجراءات مفعلة (actionEnabled = true)
→ يحتوي على DISABLE_PLATFORM
→ النظام يعيد توجيهه فوراً لصفحة /blocked
→ رسالة واضحة: "تم إيقاف المنصة - يجب سداد 15,000 ج.م"
→ لا يمكنه الوصول لأي محتوى
```

### سيناريو 3: متدرب لديه استثناء ⭐

```
الموعد العام: 31 ديسمبر (+ 7 أيام = 7 يناير)
استثناء المتدرب: موعد نهاية 28 فبراير (+ 10 أيام = 10 مارس)

المتدرب يسجل الدخول في 15 يناير:
→ النظام يستخدم موعد الاستثناء (10 مارس)
→ لم يتجاوز الموعد بعد
→ يرى banner: "متبقي 54 يوم"
→ المنصة تعمل عادياً
```

### سيناريو 4: إجراءات جزئية 🟠

```
الإجراءات المحددة:
- DISABLE_QUIZZES ✓
- DISABLE_ATTENDANCE ✓
- (بدون DISABLE_PLATFORM)

المتدرب تجاوز الموعد:
→ المنصة تعمل ✅
→ الاختبارات محجوبة 🚫
→ نظام الحضور محجوب 🚫
→ باقي الميزات تعمل
```

## جدول القرارات

| حالة الدفع | الموعد | actionEnabled | الإجراءات | النتيجة |
|------------|--------|---------------|-----------|---------|
| PAID | - | - | - | ✅ وصول كامل |
| PENDING | لم يحن | - | - | ✅ وصول كامل |
| PENDING | ضمن الفترة | true | أي | 🟡 إشعار + وصول |
| PENDING | تجاوز | false | - | 🟡 إشعار + وصول |
| PENDING | تجاوز | true | DISABLE_PLATFORM | 🔴 حجب كامل |
| PENDING | تجاوز | true | DISABLE_QUIZZES | 🟠 حجب جزئي |
| PARTIALLY_PAID | تجاوز | true | DISABLE_PLATFORM | 🔴 حجب كامل |

## ملاحظات مهمة

### ⚠️ نقاط الانتباه:

1. **actionEnabled مهم جداً**:
   - إذا كان `false`، لا تطبق أي إجراءات
   - يجب تفعيله في صفحة مواعيد السداد

2. **التواريخ**:
   - يتم المقارنة باستخدام `new Date()`
   - التأكد من المنطقة الزمنية الصحيحة

3. **الاستثناءات**:
   - `feeId = null` → يطبق على جميع الرسوم
   - `feeId = <number>` → يطبق على رسم محدد

4. **الأداء**:
   - الفحص يحدث مرة واحدة كل 5 دقائق
   - ليس في كل صفحة أو كل طلب
